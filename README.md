# VoltaVPN

VoltaVPN — open-source desktop VPN-клиент с фокусом на простоту, приватность и безопасность по умолчанию.

Проект находится в активной разработке и пока не является финальным production-релизом для массового использования.

## Идея проекта

VoltaVPN создаётся как понятный VPN-клиент без перегруженного интерфейса и сложной настройки:

- быстрый старт;
- прозрачный статус подключения;
- минималистичный UI;
- предсказуемое поведение в критичных сценариях.

Цель — сделать продукт удобным для обычного пользователя, не жертвуя инженерной строгостью в вопросах безопасности.

## Принципы

- **Security first**: чувствительные проверки работают по принципу fail-closed.
- **Privacy by default**: не собираем лишние данные.
- **Прозрачность разработки**: ключевые решения и ограничения документируются публично.
- **Практичный UX**: меньше визуального шума, больше ясности.

## Что уже реализовано

- базовый desktop-клиент на Go + Fyne;
- экранные потоки и настройки приложения;
- интеграционные точки для backend API;
- набор первичных hardening-мер:
  - HTTPS-only для API;
  - allowlist хостов в доменной зоне `*.voltavpn.com`;
  - ужесточённый парсинг auth-link;
  - запрет неявного mock в production-сценарии;
- документация по security-практикам, threat model и secure updates.

Подробности по безопасным обновлениям описаны в `secure-updates.md`.

## Быстрый старт

### Требования

- Go 1.22+
- Windows (на текущем этапе — основной сценарий)

### Локальный запуск

Используйте стандартный Go workflow: `go run` / `go build`.

Для запуска с backend API задайте:

- `VOLTA_API_BASE_URL` — HTTPS URL, который проходит политику allowlist.

Dev-переменные (только для локальной разработки):

- `VOLTA_ALLOW_MOCK_CLIENT=1` — разрешить mock-клиент при отсутствии `VOLTA_API_BASE_URL`;
- `VOLTA_API_ALLOW_ANY_HOST=1` — временно отключить host allowlist для dev-стендов;
- `VOLTA_DEV_SKIP_LOGIN=1` — пропуск экрана входа только в dev-окружении.
